How It Works

ContextGraph is a pipeline: it discovers files, routes them to extractors, resolves entities, builds a graph, and writes everything to SQLite. Here's each stage in detail.

1. File discovery

FileDiscovery walks the project directory using the include/exclude glob patterns from your config. Each matching file becomes an Artifact with a path, type, size, and SHA-256 checksum.

On re-indexing, ChecksumTracker compares the current checksum against what's stored. If unchanged, the file is skipped — this makes incremental indexing fast even on large projects.

2. Artifact type detection

ArtifactTypeDetector maps file extensions to NodeType. A .kt file becomes CodeFile, a .sql file becomes DatabaseSchema, a .pdf becomes PDF, and so on. The type determines which extractors run.

3. Extraction

IngestPipeline runs extraction with Kotlin coroutines. Each file is processed concurrently on Dispatchers.IO — multiple extractors can run in parallel across many files. Results flow through a Channel to a single consumer that writes to SQLite, avoiding lock contention.

Each extractor implements ResourceExtractor:

ResourceExtractor.kt
interface ResourceExtractor {
    val id: String
    val supportedTypes: Set<NodeType>
    suspend fun extract(artifact: Artifact, context: ExtractionContext): ExtractionResult
}

An ExtractionResult contains a list of GraphNodes, a list of GraphEdges, and back-references to the source Artifact.

4. Entity resolution

EntityResolver deduplicates nodes across files. If a Class named UserService appears as both a definition in a .kt file and a reference in a Markdown doc, they resolve to the same node with provenance from both sources.

5. Storage

All data is written to SQLite via SqliteStorageAdapter using Jetbrains Exposed as the DSL. The database schema is managed by Flyway migrations. Three key tables:

  • artifacts — one row per indexed file
  • nodes — entities extracted from files, with FTS5 index on label for full-text search
  • edges — typed, weighted relationships between nodes
  • provenance — file + line numbers tracing each node back to its source

Query layer

The QueryEngine sits on top of StorageAdapter and provides higher-level operations: BFS neighborhood expansion (via JGraphT), shortest-path finding, reverse-dependency (impact) analysis, and context bundle assembly for AI agents.

Adding a new extractor
Implement ResourceExtractor, add it to the ExtractorRegistry in Main.kt and McpServer.kt. No other changes needed.
